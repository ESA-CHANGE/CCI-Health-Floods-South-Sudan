# Uncertainty Module

This folder contains pre-check and uncertainty analysis scripts for BayFloodGEN outputs.

## Expected input data

All scripts use model outputs generated by the main pipeline:

- `../data/model_output/bayfloodgen_output.csv` (combined observed + synthetic rows)
- `../data/model_output/bayfloodgen_trace.nc` (ArviZ trace)

> Run scripts from `src/uncertainty` (or keep paths equivalent), because paths are relative.

---

## Recommended execution order

1. **`pre_uncertainty_check.py`**   
   Validate category consistency and scenario aggregation sanity checks.
2. **`check_convergence.py`**  
   Verify synthetic scenario ranges and MCMC convergence tails (R-hat / ESS).
3. **`combined_uncertainty_analysis.py`**  
   Run the main combined uncertainty workflow and generate final uncertainty products.
4. **`trace_analysis.py`** *(optional, complementary)*  
   Additional uncertainty diagnostics and visualizations from trace + synthetic scenarios.

The key requirement is that **`pre_uncertainty_check.py` and `check_convergence.py` are executed before `combined_uncertainty_analysis.py`**.

---

## Script-by-script summary

## 1) `pre_uncertainty_check.py`

### What it does

- Prints row counts and category distributions for real vs synthetic data.
- Checks facility-level flood-rate summaries by category.
- Detects category mismatches between real and synthetic metadata.
- Builds exact scenario-mean tables to inspect SAFE-category behavior.

### Input

- `../data/model_output/bayfloodgen_output.csv`

### Output

- **Console report only** (no files written).

---

## 2) `check_convergence.py`

### What it does

- Checks for impossible facility/scenario annual flood-day values.
- Summarizes facility-level annual flood-day variability (mean/std/max/cv).
- Loads ArviZ trace and reports:
  - parameters with `R-hat > 1.05`
  - parameters with `ESS_bulk < 400`
  - low-ESS composition by parameter family.

### Input

- `../data/model_output/bayfloodgen_output.csv`
- `../data/model_output/bayfloodgen_trace.nc`

### Output

- **Console report only** (no files written).

---

## 3) `combined_uncertainty_analysis.py`

### What it does

Main production uncertainty workflow (v6-compatible):

- Corrects observed flood probability via VIIRS-channel inversion.
- Computes facility-level uncertainty decomposition:
  - posterior component
  - sensor component
  - optional VIIRS-rate uncertainty propagation (MC).
- Produces decision-confidence views (exceedance and median based).
- Produces sensitivity heatmap to VIIRS error rates.

### Input

- `../data/model_output/bayfloodgen_output.csv`

### Output directory

- `../data/uncertainty/combined_uncertainty_6`

### Main output files

- `facility_uncertainty_v6.csv`
- `uncertainty_decomposition_v6.png`
- `bias_correction_v6.png`
- `sensitivity_heatmap_v6.png`
- `decision_confidence_exceedance_v6.png`
- `decision_confidence_median_v6.png`

---

## 4) `trace_analysis.py` (optional)

### What it does

Extended diagnostics using synthetic scenarios and (if available) trace:

- Scenario-count stability curves.
- Facility-level credible interval fan analysis.
- Year-sensitivity diagnostics.
- Parameter posterior and convergence diagnostics from trace.
- Monthly uncertainty fan chart and summary table.

### Input

- `../data/model_output/bayfloodgen_output.csv`
- `../data/model_output/bayfloodgen_trace.nc` *(for posterior plots; script handles missing trace with warnings)*

### Output directory

- `../data/uncertainty/trace_uncertainty`

### Typical output files

- `A_scenario_stability.png`
- `B_facility_credible_intervals.png`
- `C_t_year_sensitivity.png`
- `D1_global_parameter_posteriors.png`
- `D2_delta_per_facility.png`
- `D3_convergence_diagnostics.png`
- `F_monthly_fan_chart.png`
- `trace_uncertainty_summary_table.csv`

---

## Quick run commands

```bash
cd src/uncertainty
python pre_uncertainty_check.py
python check_convergence.py
python combined_uncertainty_analysis.py
# optional:
python trace_analysis.py
```
